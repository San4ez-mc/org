## 💬 Редагування структури/процесів + версійність (велике, обговорюємо)

Проблема: є **поточна** структура/процеси і **планована/тестова** (відділи яких ще нема, застарілі процеси, ті що тестуються; чинні керівники vs заплановані).

💬 **Пропозиція (для обговорення):**
- Кожна одиниця (відділення/відділ/посада/процес) має **статус:** `current` (діюча), `planned` (планована), `testing` (тестується), `deprecated` (застаріла).
- **Перемикач зверху:** «Тільки поточне» / «Показати все (з планованим)».
- **Візуальне розрізнення** статусів — рамкою/штрихуванням/іконкою (⚠ НЕ кольором фону, бо колір = PAEI Адізеса). Напр.: пунктирна рамка = planned, жовтий бейдж «тест» = testing, приглушено = deprecated.
- **Історія версій** одиниці (хто, коли, що змінив).
- **Інструменти редагування доступні боту** (через API): бот може створювати тестові відділи/процеси, міняти статуси.

## Мої додаткові пропозиції

- **Формати експорту:** орг.структура → SVG (векторний, масштабується) + PNG; процеси (Mermaid) → SVG/PNG нативно; звіт по компанії → PDF.
- **Редагування:** гібрид — inline для назв/ЦКП (клік→інпут), попап-форма для складних (додати посаду, призначити людину, змінити статус, долі власників).
- **Показник деталізації процесу:** рахувати по заповненості (кроки, відповідальні, коментарі, позначки автоматизації) → %.
- **Технічні логи:** окрема таблиця `event_log` (level, source=bot/api/worker, message, companyId, ts) + сторінка з фільтрами.
