# Правила генерації посадових інструкцій (#204)

> За цими правилами система збирає чернетку інструкції посади. Зміст лягає на Google Drive
> (G3), у БД — посилання (#276). Генерація детермінована зі структурованих даних; LLM
> (за наявності ключа) лише полірує формулювання, не вигадує факти.

## Структура інструкції (канонічна модель, адмінтехнологія)
1. **Шапка:** назва посади, відділ/відділення, кому підпорядковується (reportsTo).
2. **ЦКП посади** — цінний кінцевий продукт (обовʼязковий; нема ЦКП → інструкцію не
   генеруємо, спершу треба задати ЦКП).
3. **Обовʼязки/функції** — виводяться з кроків процесів, де посада відповідальна
   (дія → результат кожного кроку). Звʼязок із процесом за id (#221).
4. **Процеси, в яких бере участь** — перелік із посиланням на процес.
5. **Взаємодії** — з ким по процесах (попередній/наступний крок).
6. **Показники (статистики)** — метрики посади (з orgUnit.statistics).
7. **Права та доступи** — похідне від посади (окрема таблиця на Drive).
8. **Вимоги/кваліфікація** — заповнює людина/онбординг-агент (Ф3.5).

## Правила
- Мова лише українська.
- Факти (ЦКП, кроки) — тільки з бази, не вигадуються.
- Обовʼязки формулюються від дії: «<дія> → <результат>».
- Порожні секції позначаються «— (заповнити)», не пропускаються (видимість прогалин, #210).
- Версія інструкції росте при кожній зміні (#276 version++).
- Статус чернетки — DRAFT, поки людина не підтвердить (ACTIVE).
